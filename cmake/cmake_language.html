1 组织

CMake 输入文件用 “cmake language” 编写。该文件名为 CMakeLists.txt 或用.cmake 作为后缀。 项目中的 CMake 源文件组织为：

  • 目录 (CMakeLists.txt)
  • 脚本 (<script>.cmake)
  • 模块 (<module>.cmake)

1.1 目录(Directories)

当 CMake 处理项目源代码树时，入口点是顶级源代码目录中名为 CMakeLists.txt 的源文件。该文件可能包含整个构建规范，或使用 add_subdirectory（） 命令将子目录添加到构建中。每个由该命令添加的子目录还必须包含一个 CMakeLists.txt 文件作为该目录的入口点。CMake 会为处理的 CMakeLists.txt 所在目录 在构建树中生成相应的目录作为默认的工作和输出目录。

1.2 脚本(Scripts)

单个<script>.cmake 源文件可以通过使用带有-P 选项的 cmake（1）命令行工具以脚本模式处理。脚本模式只运行给定的 CMake 语言源文件中的命令，不会生成构建系统。它不允许定义构建目标或操作的 CMake 命令。 

# script.cmake
foreach(arg
    NoSpace
    Escaped\ Space
    This;Divides;Into;Five;Arguments
    Escaped\;Semicolon
    )
  message("${arg}")
endforeach()

cmake -P script.cmake

1.3 模块(Modules)

目录或脚本中的 CMake 语言代码可以使用 include（） 命令在包含上下文的范围内加载<module>.cmake 源文件。有关模块的文档，请参阅 CMake 分发中包含的 cmake-modules（7）手册页。项目源代码树也可以提供它们自己的模块，并在 CMAKE_MODULE_PATH 变量中指定它们的位置。

2 语法（Syntax）

2.1 编码（Encoding）

CMake 语言源文件可以用 7 位 ASCII 文本编写，以便在所有支持的平台上实现最大的可移植性。换行符可能被编码为“\n” 或“\r \n”，但作为输入文件被读取时会转换为“\n”。

请注意，该实现是 8 位清理的，因此如果平台系统 API 支持，源文件将被编码为 UTF-8。另外，CMake 3.2 及以上版本支持在 Windows 上使用 UTF-8 编码的源文件（使用 UTF-16 调用系统 API）。此外，CMake 3.0 及更高版本允许在源文件中使用首部 UTF-8 字节顺序(Byte-Order Mark) 标记。

2.2 源文件（Source Files）

CMake 语言源文件由零个或多个命令调用(Command Invocations) 组成，这些命令调用由换行符和可选的空格和注释 分开。

请注意，任何不在“命令参数(Command Arguments)”或“括号注释(Bracket Comment) ”中的源文件行都可以以行注释(Line Comment) 结束。

2.3 命令调用（Command Invocations）

命令调用是一个名称，后跟括号，括号中是由空格分隔的参数。例如: 

add_executable(hello world.c)

命令名称不区分大小写。参数中嵌套的不带引号的括号必须平衡。每个命令调用中的（或）都是字面值。调用 if（）命令这可用于来封装条件。例如： 

if(FALSE AND (FALSE OR TRUE)) # evaluates to FALSE

2.4 命令参数（Command Arguments）

命令调用（Command Invocations）中有三种类型的参数：

2.4.1 括号参数（Bracket Argument）

括号参数，由 Lua 长括号语法启发，将内容封闭在相同长度的开始和结束“括号”之间： 

message([=[
This is the first line in a bracket argument with bracket length 1.
No \-escape sequences or ${variable} references are evaluated.
This is always one argument even though it contains a ; character.
The text does not end on a closing bracket of length 0 like ]].
It does end in a closing bracket of length 1.
]=])

一个左括号是写[后跟 0 或更多=后面是[。相应的右括号写为]，后跟相同的数字=，后跟]。括号不嵌套。可以选择独特的长度左右括号来包含其他长度的右括号。

括号参数内容由开始和结束括号之间的所有文本组成，紧接在开头括号之后的一个换行符（如果有的话）被忽略。对括号参数内容（例如转义序列或变量引用）不进行计算。括号参数总是作为一个参数赋给命令调用。

2.4.2 引号参数（Quoted Argument）

引用参数(Quoted Argument)包含在双引号("")中间。

引用参数内容由开始和结束引号之间的所有文本组成。转义序列（Escape Sequences） 和变量引用（Variable References）。引用参数总是作为命令调用的一个参数。例如： 

message("This is a quoted argument containing multiple lines.
This is always one argument even though it contains a ; character.
Both \\-escape sequences and ${variable} references are evaluated.
The text does not end on an escaped double-quote like \".
It does end in an unescaped double quote.
")

任何行上以奇数个反斜杠结尾的的最后那个“\”被视为一个续行，它和紧跟在后面的换行符将被忽略。例如： 

message("\
This is the first line of a quoted argument. \
In fact it is the only line but since it is long \
the source code uses line continuation.\
")

2.4.3 非引号参数（Unquoted Argument）

非引号参数不包含在任何引用语法中。它可能不包含任何空格，（，），＃，“或\，除非被反斜线转义。 非引号参数内容由允许或可转义的字符的连续块中的所有文本组成。转义序列和变量引用都会被求值。结果值以列表分成元素的相同的方式进行切分。每个非空元素都作为参数赋给命令调用。因此，一个非引号参数可以作为零个或多个参数传给命令调用。 

foreach(arg
    NoSpace
    Escaped\ Space
    This;Divides;Into;Five;Arguments
    Escaped\;Semicolon
    )
  message("${arg}")
endforeach()

2.5 转义序列（Escape Sequences）

转义序列是“\”后跟一个字符。

“\”后面跟着一个非字母数字字符，只是对字面字符进行编码，而不会将其解释为语法。“\t”，“\r”或“\n”分别对制表符，回车符或换行符进行编码。“\;”用在任何变量引用之外可以编码变量本身，但用在未引用的参数中来编码“;”，而不是分割参数值。变量引用内部的“;”用来编码字面量的“;”字符。

2.6 变量引用（Variable References）

变量引用的格式为$ {variable_name}，并在引用的参数或非引用的参数中进行求值。变量引用被变量的值替换，或者如果变量没有被设置，则由空字符串替换。变量引用可嵌套，并从内向外进行评估，例如， $ {_outer_${inner_variable}_variable} 。

字面变量引用可能由字母数字，/_.+-和转义序列组成。可以使用嵌套引用来对任何名称的变量进行求值。

变量（variables） 部分记录变量名称的范围以及如何设置它们的值。

环境变量引用的形式为 $ENV{VAR} ，并在与普通变量引用一样在上下文中进行求值。

2.7 注释（Comments）

2.7.1 括号注释（Bracket Comment）

“#”紧接着一个括号参数（Bracket Argument）构成了一个由整个括号框组成的括号注释，例如： 

#[[This is a bracket comment.
It runs until the close bracket.]]
message("First Argument\n" #[[Bracket Comment]] "Second Argument")

2.7.2 行注释（Line Comment）

“#”不立即跟着一个括号参数(Bracket Argumen)形成一个行注释直到行尾，例如： 

# This is a line comment.
message("First Argument\n" # This is a line comment :)
        "Second Argument") # This is a line comment.

3 控制结构（Control Structures）

3.1 条件块（Conditional Blocks）

if()/elseif()/else()/endif()命令分隔代码块以有条件地执行。

3.2 循环（Loops）

foreach()/endforeach()和 while()/endwhile()命令分隔代码块，以循环执行。在这样的块内部，break()命令可以用来提前终止循环，而continue()命令可以用来立即开始下一次迭代。

3.3 命令定义（Command Definitions）

macro()]]/endmacro()和 function()/ endfunction()命令将代码块分隔以便稍后调用以作为命令被记录。

4 变量（Variables）

变量是 CMake 语言的基本存储单元。它们的值始终是字符串类型，尽管有些命令可能会将字符串解释为其他类型的值。set()和 unset()命令明确设置或取消设置变量，但其他命令也具有修改变量的语义。变量名称区分大小写，几乎可以包含任何文本，但是我们推荐使用只包含字母数字字符加上_和 - 的名称。

变量具有动态范围。每个变量“set”或“unset”在当前范围内创建一个绑定：

  • 函数作用域（Function Scope）

    由 function（）命令创建的命令定义创建命令，当被调用时，在一个新的变量绑定范围中处理记录的命令。通过“set”或“unset”在此范围内绑定的变量，只对于当前函数及其内部的任何嵌套调用可见，函数返回后不可见。

  • 目录作用域（Directory Scope）

    源树中的每个目录都有自己的变量绑定。在处理目录中的 CMakeLists.txt 文件之前，CMake 会复制父目录中当前定义的所有变量绑定（如果有的话）以初始化新的目录作用域。当用 cmake -P 进行处理时，CMake 脚本将变量绑定在一个“目录”范围内。

  • 持久缓存（Persistent Cache）

    CMake 存储一组单独的“缓存”变量或“缓存条目”，它们的值在项目构建树中的多个运行中保持不变。缓存条目具有独立绑定范围，仅由显式请求修改，例如由 set（）和 unset（）命令的 CACHE 选项 修改。

当评估变量引用时，CMake 首先搜索函数调用堆栈（如果有的话）进行绑定，然后回退到当前目录范围中的绑定（如果有的话）。如果找到“set”绑定，则使用它的值。如果找到“unset”绑定，或者没有找到绑定，CMake 就会搜索一个缓存条目。如果找到缓存条目，则使用其值。否则，变量引用计算为空字符串。

cmake-variables(7)手册记录了由 CMake 提供的许多变量，或者在由项目代码设置时对 CMake 有意义。

5 列表（List）

尽管 CMake 中的所有值都以字符串的形式存储，但是在特定的上下文中（例如在求值一个 Unquoted Argument 期间），可能会将字符串视为列表。在这样的上下文中，一个字符串用“;”字符，将[和]不相等或“\”之后的字符分割为列表元素。序列“\;”不划分字符，而是在产生的元素被“;”取代。

元素列表通过由“;”将分隔的元素连接表示为一个字符串。例如，set（）命令将多个值作为列表存储到目标变量中： 

set(srcs a.c b.c c.c) # sets "srcs" to "a.c;b.c;c.c"

列表适用于简单用例，如源文件列表，不应用于复杂的数据处理任务。构造列表的大多数命令不会转义列表元素中的“;”的字符，从而扁平化嵌套列表： 

set(x a "b;c") # sets "x" to "a;b;c", not "a;b\;c"
