1 概述

这里主要通过一个小项目实践 cmake 在工程项目中的主要用法。主要是基于cmake-tutorial 做的一个小扩展。

2 项目结构

遵循模块话的项目结构，工程目录如下： 

├── build
├── CMakeLists.txt
└── src
  ├── hello
  │   ├── hello.cpp
  │   └── include
  │       └── hello
  │           └── hello.h
  └── test
      └── test.cpp

2.1 src

所有的源代码放在 src 目录下。

hello.h 是 hellolib 的头文件： 

#include <string>

void say_hello(std::string name);

hello.cpp 实现和 say_hello 函数： 

#include <iostream>

void say_hello(std::string name) {
		std::cout<<"hello "<<name << std::endl;
}

test.cpp 调用了 say_hello 函数： 

#include <iostream>

#include <hello/hello.h>

int main(int argc, char *argv[])
{
		say_hello("John");
		return 0;
}

2.2 build

为了保持源代码目录干净，build 目录用于编译和打包代码。

3 cmake V1

最外层的 CMakeLists.txt 来编译整个项目。 

cmake_minimum_required(VERSION 3.0)
project(cmake_test VERSION 0.2)

add_library(hello src/hello/hello.cpp)

add_executable(cmake_test src/test/test.cpp)
target_link_libraries(cmake_test hello)
target_include_directories(cmake_test PUBLIC
  /home/lsl/github/test/cmake/cmake_dir/src/hello/include)

4 cmake V2

其实 hello 库应该有自己独立的 cmake 文件，这样有几个好处：

  • cmake 文件也是模块化组织的。
  • 方便 hello 库被其他程序集成编译。

所以新的代码目录如下： 

├── build
├── CMakeLists.txt
└── src
    ├── hello
    │   ├── CMakeLists.txt
    │   ├── hello.cpp
    │   └── include
    │       └── hello
    │           └── hello.h
    └── test
        ├── CMakeLists.txt
        └── test.cpp

src/CMakeLists.txt 

cmake_minimum_required(VERSION 3.0)
project(cmake_test VERSION 0.2)
add_subdirectory(src/hello)
add_subdirectory(src/test)

src/hello/CMakeLists.txt 

add_library(hello hello.cpp)

src/test/CMakeLists.txt 

add_executable(cmake_test test.cpp)
target_link_libraries(cmake_test hello)
target_include_directories(cmake_test PUBLIC
  /home/lsl/github/test/cmake/cmake_dir/src/hello/include)

在 build 下运行 cmake 和 make，正如 cmake language 中所说：“CMake 会为处理的 CMakeLists.txt 所在目录在构建树中生成相应的目录作为默认的工作和输出目录”。 

build$ ls src/hello/
CMakeFiles  cmake_install.cmake  libhello.a  Makefile
build$ ls src/test/
CMakeFiles  cmake_install.cmake  cmake_test  Makefile

5 cmake V3

src/test/CMakeLists.txt 中为了导入 hello 库的模块，使用了绝对路径： 

target_include_directories(cmake_test PUBLIC
  /home/lsl/github/test/cmake/cmake_dir/src/hello/include)

这样不利于项目路径的迁移。

src/test/CMakeLists.txt 

add_executable(cmake_test test.cpp)
target_link_libraries(cmake_test hello)

add_library(hello hello.cpp)
target_include_directories(hello PUBLIC
  INTERFACE include)

正如 cmake build system 中所说：“依赖项的使用需求的是可以传递的”：

The usage requirements of a target can transitively propagate to dependents.

hello 库 INTERFACE_INCLUE_DIRECTORIES 目标属性值将会传递复制到到 test 的 INCLUDE_DIRECTORiES 中，所有 target 也能找到 hello/hello.h 头文件。
