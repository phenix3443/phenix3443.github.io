如何使用hexo与github搭建个人静态博客

只是使用wordpress在空间上写了一段时间的博客，但是后来感觉域名购买比较坑，自己写的也比较少，加上想试试markdown，就想着在github建立了一个静态博客就已经够用了。

同时备选的有hexo与jelly，但是同学说hexo简单一点，所以就先试试这个，果然安装比较方便。

这篇博客主要是用来介绍这个静态网站从搭建到优化后面一步步的过程，所以也注定是持续更新的。

  • GitHub上建立一个名为phenix3443.github.io的repo，其中phenix3443是自己的用户名，github规定每个用户只能建一个这样的repo。
  • 根据Hexo官方文档进行安装
  • mkdir -p ~/projects/hexo
  • hexo init
  • hexo g
  • hexo s #启动hexoserver
  • 打开localhost:4000就能看到刚才写的第一篇博客了

接下来就是要把博客内容发布（deploy）github上面，需要在hexo的配置文件中进行设置

  • 将_config.yml文件Deployment字段设置如下：
    deploy：
    type: git
    repo: git@github.com:phenix3443/phenix3443.github.io.git
    branch: master
  • 保证本机生成了ssh-key，并在github进行了注册。
  • hexo d #发布
  • 浏览器打开phenix3443.github.io就能看到文件了。
  • 注意，post文件要使用utf-8进行编码，要不容易出现乱码。
