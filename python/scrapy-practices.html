1 概述

本文通过实践，逐步学习爬虫知识。文中代码如无特殊说明都来自Scrapy 1.5 documentation，

使用爬虫主要是为了方便的从网络上获取一些想要的数据，在此过程中，涉及到数据的抓取、过滤、分析和存储，我们可以使用一些基础的 python 库，如 requesets、beautifulsoup 来完整实现实现爬虫。但也可以使用现成的框架，如 scrap，好处在于，它：

  1. 抽象爬虫的工作流程，实现了很多通用的模块，如下载模块，解析模块。我们只要关注核心的业务模块就可以了。
  2. 提供了很多内置的功能，例如内置的几种 export 模块。
  3. 可以高性能、分布式的爬取信息。scrapy 基于 twisted 实现了异步请求，可以异步处理爬虫任务。

2 最简单的爬虫

前面说到，使用 scrapy，我们只需要关注核心的业务模块就可以。爬虫的整个工作流程中最基础的就是页面的抓取和分析了，下面我们来实现一个最简单的爬虫： 

import scrapy


class QuotesSpider(scrapy.Spider):
    name = "quotes"
    start_urls = [
        'http://quotes.toscrape.com/tag/humor/',
    ]

    def parse(self, response):
        for quote in response.css('div.quote'):
            yield {
                'text': quote.css('span.text::text').extract_first(),
                'author': quote.xpath('span/small/text()').extract_first(),
            }

        next_page = response.css('li.next a::attr("href")').extract_first()
        if next_page is not None:
            yield response.follow(next_page, self.parse)

这是我们通过类实现的一个爬虫，具体的语法内容参考文档，下面我们运行该爬虫： 

scrapy runspider quotes_spider.py -o quotes.json

scrapy 可以单独运行一个爬虫，通过上面的代码：

  • 从网页中提取两个内容：格言的内容（text）和作者（author）
  • next_page 表示继续爬取下一页内容
  • -o~参数表示将提取的内容保存到 ~quotes.json 文件中。

3 爬虫项目

一个爬虫项目当然不可能只是上面一个爬虫类，下面我们看下如何建立一个工程意义上的爬虫项目： 

scrapy startproject tutorial

scrapy 提供很多命令行工具 ，可以方便的建立爬虫项目、改变项目设置、调试爬虫等。上面的命令建立了一个名为 tutorial 的爬虫项目。tutorial 下的文档结构： 

tutorial/
  scrapy.cfg            # deploy configuration file
  tutorial/             # project's Python module, you'll import your code from here
      __init__.py
      items.py          # project items definition file
      middlewares.py    # project middlewares file
      pipelines.py      # project pipelines file
      settings.py       # project settings file
      spiders/          # a directory where you'll later put your spiders
          __init__.py

之后我们所有的爬虫都会放在 tutorial/spiders 目录下。 将之前写的 quote_spider.py 放在此目录下。现在我们运行完整的爬虫项目： 

scrapy crawl quotes -o quotes.json

注意：输出是追加到文件中，它并不会清空原先 quotes.json 中的内容。

查看其中的内容，可以发现最终的数据都被放在 json 数组中，我们更希望每条数据一行记录： 

scrapy crawl quotes -o quotes.jl

5 解析页面元素

爬取的关键在于如何解析页面中的元素，这涉及到 scrapy 中药的概念：选择器（selectors）。选择器提供 xpath 和 css 两种方式来选取页面元素，参见 Selectors。 实践中，我们通常结合两种方式，快速的选定元素。

6 存储结果

上面的步骤中，parse 函数中，将解析到的数据组成 python dict 结构，通过 yield 返回，最终保存在 json 文件中。

但实际工作中，对数据的处理并不是这么简单，这些数据会进行过滤、分析、然后可能会被发送到其他服务器，存储到数据库等。scrapy 中通过 Pipeline （管道）来实现这样的流程，（有点像 linux 的管道操作符）。在 pipeline 中传输的数据被抽象为Items。

在 tutorial/tutorial/items.py 中添加自定的 item： 

class QuoteItem(scrapy.Item):
    # store quotes spider result
    text = scrapy.Field()
    author = scrapy.Field()

修改 tutorial/tutorial/spiders/quotes_spider 中响应的代码： 

# -*- coding: utf-8 -*-
import scrapy
from tutorial.items import QuoteItem # 注意此处的引用



class QuotesSpider(scrapy.Spider):
    name = 'quotes'
    # custom_settings = {
    #     'ITEM_PIPELINES': {
    #         'tutorial.pipelines.CSVPipeline': 400
    #     }
    # }
    allowed_domains = ['quotes.toscrape.com']
    start_urls = ['http://quotes.toscrape.com/page/1/']

    def parse(self, response):
        for quote in response.css('div.quote'):
            qi = QuoteItem()
            qi["text"] = quote.css('span.text::text').extract_first()
            qi["author"] = quote.css('small.author::text').extract_first()
            yield qi

        next_page = response.css('li.next a::attr(href)').extract_first()
        if next_page is not None:
            yield response.follow(next_page, callback=self.parse)

在 tutorial/tutorial/pipelines.py 中添加自定义的管道： 

class CSVPipeline(object):
    def open_spider(self, spider):
        self.csv_file = open("quotes.csv", "w")

    def close_spider(self, spider):
        self.csv_file.close()

    def process_item(self, item, spider):
        if spider.name == "quotes":
            text = item.get("text")
            author = item.get("author")
            self.csv_file.write("{},{}\n".format(text, author))
            return item

CSVPipeline 将 itemb 保存到 csv 文件中。

注意： process_item 最后要将 item 返回 ，需要注意这样一个事实：一个 item 是可以经过多个 item 处理的。

现在修改设置，向 scrapy 框架注册 pipeline，这样 scrapy 才可以调用 pipeline 处理 item。

修改 tutorial/tutorial/settings.py： 

ITEM_PIPELINES = {
    'tutorial.pipelines.TutorialPipeline': 300,
    'tutorial.pipelines.CSVPipeline': 400
}

pipeline 后面的数据表示优先级，数字越小，优先级越高。

7 控制 spider 数据流向

实际项目中不可能只有一个爬虫， 一般来说， 一个有效的页面解析对应一个爬虫 。我们添加一个解析格言作者信息的爬虫：

tutorial/tutorial/spiders/author_spider.py 

# -*- coding:utf-8 -*-
# author:liushangliang@xunlei.com
import scrapy


class AuthorSpider(scrapy.Spider):
    name = 'author'

    start_urls = ['http://quotes.toscrape.com/']

    def parse(self, response):
        # follow links to author pages
        for href in response.css('.author + a::attr(href)'):
            yield response.follow(href, self.parse_author)

        # follow pagination links
        for href in response.css('li.next a::attr(href)'):
            yield response.follow(href, self.parse)

    def parse_author(self, response):
        def extract_with_css(query):
            return response.css(query).extract_first().strip()

        yield {
            'name': extract_with_css('h3.author-title::text'),
            'birthdate': extract_with_css('.author-born-date::text'),
            'bio': extract_with_css('.author-description::text'),
        }

现在有两个 spider，他们解析到了不同的数据，如何控制各自 spider 产生的数据经过不同 pipeline 呢？有两种方法：

  • 方法一通过 pipeline 中的 spider.name 来做判断。
  • 方法二通过在 spdier 中的 custom_settings 来指定。 
custom_settings = {
        'ITEM_PIPELINES': {
            '<project_name>.pipelines.<pipeline_name>': 100
        }
    }

实践中：通常用第二种方法指定 spider 特定的 pipeline。在 settings.py 中制定所有 spider 都要经过的 pipeline。

现在只需要 quotes_spider 将输出写到 csv 文件，修改后 quotes_spider.py： 

# -*- coding: utf-8 -*-
import scrapy
from tutorial.items import QuoteItem


class QuotesSpider(scrapy.Spider):
    name = 'quotes'
    custom_settings = {
        'ITEM_PIPELINES': {
            'tutorial.pipelines.CSVPipeline': 400
        }
    }
    allowed_domains = ['quotes.toscrape.com']

9 部署

可以通过定时任务让爬虫程序定时执行。但是如果有多个爬虫，就需要并行执行来提高效率。

scrapy 提供了更好的方式来部署爬虫：scrapyd。 scrapyd 是 demon 程序，提供了 HTTP 接口对爬虫进行方便的部署、运行和监控。。 

sudo pip3 install scrapyd

但是scrapyd-client 提供了更方便的操作方式。 

sudo pip3 install --upgrade git+https://github.com/scrapy/scrapyd-client

注意：使用 pip3 install scrapyd-client 安装的程序将会在命令行找不到 scrapyd-client 命令，所以需要用 github 代码进行更新升级。

现在进入 scrapy.cfg 目录下（爬虫的 root 目录）： 

tutorial$ scrapyd-deploy

scrapyd 接口返回如下信息： 

Packing version 1517472918
Deploying to project "tutorial" in http://localhost:6800/addversion.json
Server response (200):
{"project": "tutorial", "spiders": 2, "node_name": "lsl-ThinkPad-X1-Carbon-5th", "status": "ok", "version": "1517472918"}

上述信息说明：名为 tutorial 的 project 部署成功，它有两个 spider，具体看下： 

tutorial$ scrapyd-client spiders -p tutorial

正式代码中实现的连个 spider。 

tutorial:
  author
  quotes

运行 quotes 的爬虫： 

tutorial$ scrapyd-client schedule -p tutorial quotes

scrapyd-client 并没有完整的支持 scrapyd 所有 HTTP 接口，比如查询爬虫进度的接口： 

curl http://localhost:6800/listjobs.json?project=tutorial | python -m json.tool
